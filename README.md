# Devart MCP Server for PostgreSQL

Self-hosted MCP server for secure AI access to PostgreSQL.

Devart MCP Server for PostgreSQL is a middleware component that sits between your AI tool and PostgreSQL through the corresponding Devart ODBC Driver. With on-premises deployment, it helps keep data inside your infrastructure and reduces the need to build and maintain a custom MCP integration from scratch.

[⬇ Download Devart MCP Server for PostgreSQL](https://www.devart.com/mcp/postgresql/) | [Documentation](https://docs.devart.com/)

## Why Devart MCP Server for PostgreSQL

Use a focused MCP server for PostgreSQL instead of building and maintaining a custom integration layer for your AI tools.

Devart MCP Server for PostgreSQL is designed for teams that need practical, secure AI-to-data access with simpler setup, source-specific behavior, and trusted Devart connectivity underneath.

## How it works

Devart MCP Server for PostgreSQL sits between your AI tool and your PostgreSQL database.

![Devart MCP Server architecture](https://github.com/devart-ai-connectivity/.github/blob/main/assets/mcp-server-architecture.png)

1. An AI tool sends a natural-language request to the MCP server.
2. The MCP server processes the request and generates SQL.
3. The Devart ODBC Driver for PostgreSQL translates and executes the request against PostgreSQL.
4. PostgreSQL returns the data.
5. The MCP server transforms the result into output suitable for AI processing.

## Quick start

1. Install Devart MCP Server for PostgreSQL.
2. Install the required [Devart ODBC Driver for PostgreSQL](https://www.devart.com/odbc/postgresql/).
3. Configure your PostgreSQL connection in the product UI.
4. Connect your AI tool using the integration guide for your tool.
5. Run your first natural-language query.
